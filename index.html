Goal

We want to be able to collect collaboration signals for hackathons by building an online hackathon tool that supports project workspaces. During work this spring we built the basic framework for project disclosure, reviews, and event summarization but in order to obtain data on individual and project performance as well as group collaboration we would need workspaces that support online meetings (Google Hangouts), chat (IRC), and code (git and pull request discussions).

This would focus effort directly on running hackathons that are completely virtual rather than the in person hackathons that are currently supported by applications. However, for this summer some of the core functionality that would need to be present for large-scale deployment (dynamic scheduling, initial project vetting, and team formation recommendations) would not be a top priority. The focus would be on prototyping the tools for collecting and analysing the data on collaboration from the workspace.

The long-term goal of this effort is triggering system processes for automated resource allocation and collaboration at large scale.

Keywords: hackathons, group dynamics, microtasking
